# Aone

> Conversational operations agent for freelancers and SMBs. Processes your Gmail messages and answers business questions in natural language right from the terminal.

## What it does

```bash
$ aone ask "how much does Acme owe me?"
Acme owes you $3,450 USD across 3 invoices:
  • Invoice #1024 ($1,200) overdue by 15 days
  • Invoice #1031 ($1,500) due in 5 days
  • Invoice #1042 ($750) due in 20 days

$ aone ask "summarize my conversations with Acme this month"
[summary generated by the model based on your actual emails]

$ aone ask "which clients haven't replied to me in over 30 days?"
[prioritized list of inactive clients]
```

The agent is multilingual — ask in English, Spanish, or whatever the underlying model speaks. The codebase and docs are English-only.

## Stack

v0 runs **100% free** by default, but the architecture is provider-agnostic: switching to paid models (Claude, GPT, Gemini) is just editing `.env` (see [ADR-005](./docs/decisions/005-model-agnostic-stack.md)).

- **Python 3.12** + **uv** for dependency management
- **LangGraph** for agent orchestration
- **LiteLLM** as the model wrapper (routes by provider based on env vars)
- **Groq** (free tier) — `llama-3.3-70b-versatile` for generation, `llama-3.1-8b-instant` for classification
- **`sentence-transformers/all-MiniLM-L6-v2`** local for embeddings (zero cost, runs on CPU)
- **FAISS** for in-memory semantic search
- **Gmail API** for email access
- **Langfuse Cloud** (free tier, 50k events/month) for tracing and observability

### Switching to paid models (optional)

```bash
# .env — example with Claude + OpenAI
AONE_MODEL_GENERATION=anthropic/claude-haiku-4-5
AONE_MODEL_CLASSIFICATION=openai/gpt-4o-mini
AONE_EMBEDDING_PROVIDER=litellm
AONE_EMBEDDING_MODEL=openai/text-embedding-3-small
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
```

No rebuild, no code change.

## Architecture

```
┌──────────────────────────────────────────────┐
│  CLI (Typer)                                 │
└────────────────────┬─────────────────────────┘
                     ↓
┌──────────────────────────────────────────────┐
│  Agent (LangGraph state machine)             │
│    classify_intent → select_tools →          │
│    execute_tools → generate_response         │
└────────────────────┬─────────────────────────┘
                     ↓
┌──────────────────────────────────────────────┐
│  Data layer (in-memory + pickle)             │
│    EmailCache (dict) + FAISS index           │
└─────────────────────┬────────────────────────┘
                     ↓ periodic sync
┌──────────────────────────────────────────────┐
│  Gmail API (OAuth desktop)                   │
└──────────────────────────────────────────────┘
```

Deliberately no database in v0. All persistence is local (pickle + JSON). The Postgres migration path is documented in `docs/decisions/`.

## Quickstart

### Prerequisites
- Python 3.12+
- [uv](https://docs.astral.sh/uv/) installed
- Free API key from [Groq](https://console.groq.com)
- Free account on [Langfuse Cloud](https://cloud.langfuse.com) (public + secret key)
- OAuth credentials from [Google Cloud Console](https://console.cloud.google.com) (Gmail API enabled)

### Setup

```bash
# Clone
git clone https://github.com/Axelcv24/aone.git
cd aone

# Install dependencies
uv sync

# Configure environment variables
cp .env.example .env
# edit .env with your keys

# Put your Google credentials.json in the repo root

# First sync (opens browser for OAuth on first run)
uv run aone sync

# Ask your first question
uv run aone ask "how many emails do I have from the last week?"
```

### Commands

```bash
uv run aone sync              # sync the last N Gmail messages
uv run aone ask "question"    # query the agent
uv run aone stats             # local cache statistics
uv run aone evals             # run the evaluation suite
```

## Technical decisions

Documented as ADRs in `docs/decisions/`:

- **001**: No DB in v0, memory + pickle only
- **002**: FAISS in-memory over Pinecone/Qdrant for v0
- **003**: LangGraph over flat LangChain
- **004**: MCP-ready architecture for tools
- **005**: Model-agnostic stack (LiteLLM + env vars). Free defaults, swap to paid without touching code
